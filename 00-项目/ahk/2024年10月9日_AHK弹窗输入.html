2024-10-09 11:26 "AHK 如何设计弹窗 "

  • AHK 进行窗口设计，需要网页设计么 📅 2024-10-09 ✅ 2024-10-09

在 AutoHotkey 中，您可以使用 MsgBox 命令来创建一个简单的弹窗。如果您使用的是 AutoHotkey 2.0，语法有些变化。以下是 AutoHotkey 1.0 和 2.0 版本中创建弹窗的示例。
[tab]\

AutoHotkey 1.0 自定义弹窗：

MsgBox, 36, 标题, 这是带有是/否按钮的弹窗示例。
IfMsgBox Yes
{
    MsgBox, 您点击了是。
}
Else
{
    MsgBox, 您点击了否。
}

AutoHotkey 2.0 自定义弹窗：

response := MsgBox("这是带有是/否按钮的弹窗示例。", "标题", "YN")
if response = "Yes"
{
    MsgBox "您点击了是。"
}
else
{
    MsgBox "您点击了否。"
}

MsgBox 指令（WIN）用于调用系统弹窗，如上，第一个参数是窗内文字，第二个是标题，第三个是按钮类型。

常用按钮选项

  • Y：表示“是”按钮。
  • N：表示“否”按钮。
  • O：表示“确定”按钮。
  • C：表示“取消”按钮。
  • A：表示“重试”按钮。
  • I：表示“忽略”按钮。
  • B：表示“中止”按钮。

2024-10-09 11:32 " 弹窗如何进行输入 "

使用 InputBox 函数，同样也是第三个参数作为选项（options）；第四个参数用于设置超时时间。

  • timeoutSeconds := 10：定义一个变量 timeoutSeconds 并设置其值为 10，表示输入框的超时时间为 10 秒。

:::info
使用 Value 属性返回输入值，InputBox 的属性有几个。
:::

密码样式（隐蔽）：

response := InputBox "请输入您的密码：", "输入示例", "Password"
if ErrorLevel  ; 检查用户是否点击了取消按钮。
{
    MsgBox "您取消了输入。"
}
else
{
    MsgBox "您输入的密码是：" .response.Value
}

ErrorLevel 的问题

ErrorLevel 没有全局声明，实际上 global ErrorLevel := -1 在前面声明就好了，也许是单独的文本编辑器的问题。

2024-10-09 11:50 " 除 InputBox 外还有什么接受输入的方式 "

在 AutoHotkey (AHK) 中，除了 InputBox 之外，还有几种其他方式可以接受用户输入。以下是几种常见的输入方法：

  • AHK 如何通过命令行传参 📅 2024-10-09 ✅ 2024-10-09
  1. 命令行参数：如果你的脚本是通过命令行启动的，你可以直接从 %0% 到 %9% 获取命令行参数。%0% 表示传递给脚本的参数数量，%1% 到 %9% 分别表示第一个到第九个参数。
  • AHK 如何进行 GUI 设计 📅 2024-10-09 ✅ 2024-10-09
  1. Gui：你可以创 建一个图形用户界面（GUI）来接受用户输入。下面是一个简单的示例，展示了如何创建一个包含编辑框的 GUI 来获取用户输入：
Gui, Add, Edit, vUserInput, Enter your input here
Gui, Add, Button, gSubmit, Submit
Gui, Show,, Input Example
return

Submit:
Gui, Submit
MsgBox You entered: %UserInput%
return

GuiClose:
ExitApp
  1. FileRead：如果你希望用户从文件中提供输入，可以使用 FileRead 命令来读取文件内容。例如：
FileRead, UserInput, C:\path\to\your\file.txt
MsgBox The file contents are:`n%UserInput%
  1. ClipBoard：用户可以通过剪贴板提供输入。你可以使用 ClipWait 命令等待剪贴板内容，然后使用 Clipboard 变量获取内容。例如：
MsgBox Please copy your input to the clipboard, then press OK.
ClipWait
MsgBox You copied:`n%Clipboard%
  • AHK 输入：COM 对象和外部应用程序 📅 2024-10-09 ✅ 2024-10-09
  1. COM 对象和外部应用程序：通过使用COM 对象或与其他应用程序（如 Excel、Word 等）交互，你可以从这些应用程序中获取数据。

每种方法都有其适用的场景，选择哪种方法取决于你的具体需求和应用场景。

" 标签 "

在 AutoHotkey 中，Submit 和 GuiClose 并不是传统意义上的“函数”，而是被称作“标签”（Label）。标签在 AHK 中用于定义脚本的特定行为，例如响应用户操作或处理特定事件。

在你的 GUI 示例中：

  • Submit: 是一个标签，当用户点击“Submit”按钮时，脚本会跳转到这个标签并执行其后的代码。这是一种处理按钮点击事件的方式。
  • GuiClose: 也是一个标签，它定义了当 GUI 窗口被关闭时应该执行的操作。在这个例子中，GuiClose: 标签内的代码会退出应用。

以下是你的 GUI 示例，稍微修改以更清楚地展示标签的作用：

; 创建GUI窗口，添加一个编辑框和一个按钮
Gui, Add, Edit, vUserInput, Enter your input here
Gui, Add, Button, gSubmit, Submit
Gui, Show,, Input Example
return

; Submit标签：处理按钮点击事件
Submit:
Gui, Submit  ; 将GUI中的输入数据提交到相应的变量中
MsgBox You entered: %UserInput%
return

; GuiClose标签：处理GUI窗口关闭事件
GuiClose:
ExitApp  ; 退出应用程序

在这个脚本中：

  • Gui, Add, Edit, vUserInput, Enter your input here：添加一个编辑框到 GUI，并将其值绑定到变量 UserInput。
  • Gui, Add, Button, gSubmit, Submit：添加一个按钮，当点击时跳转到 Submit 标签。
  • Gui, Show,, Input Example：显示 GUI 窗口。
  • Submit: 标签定义了按钮点击事件的处理逻辑。
  • GuiClose: 标签定义了 GUI 窗口关闭事件的处理逻辑。

这种使用标签来处理事件的方式是 AutoHotkey 中常见的一种编程模式。
